# v2 Model Plan

**Status:** draft — decide on the changes, then implement
**Date:** 2026-04-24
**Previous:** v1.1 (lstm=256, 633K params) plateaued at val_cer ~0.47

## Why v1.1 plateaued

Three diagnosed causes:

1. **MobileNetV3-Small stem is wrong for OCR.** It's an ImageNet classifier backbone, designed to compress spatial information into abstract class labels. OCR needs the opposite — preserve fine positional + character-shape detail. Cutting MobileNetV3 at stride 4 only surfaces 40 channels — thin features to feed a BiLSTM that needs to distinguish 44 different glyphs.

2. **Crop is wider than it needs to be, bringing in artist-name noise.** Bottom-left region `(0, 0.90, 0.50, 1.00)` captures the left half of the bottom 10%. On modern cards the artist credit often bleeds into the right edge of that region, adding text the model has to learn to *ignore* — eating model capacity.

3. **One BiLSTM layer isn't enough sequence modeling.** A single BiLSTM learns local patterns well but struggles to combine features across many timesteps (e.g. "this is a 3-digit number followed by a letter, so the letter is a rarity code"). Two stacked BiLSTMs let higher layers reason about larger context.

## v2 architecture changes

All three fixes together:

### 1. Replace MobileNetV3-Small with a purpose-built OCR stem

Custom CNN, roughly:

```
Input (48, 256, 3) uint8
  → Rescaling to [-1, 1]
  → Conv 3x3, s=(2,2), 32c  → (24, 128, 32)
  → Conv 3x3, s=(2,2), 64c  → (12, 64, 64)
  → Conv 3x3, s=(2,1), 128c → (6, 64, 128)   # halve H only
  → Conv 3x3, s=(2,1), 192c → (3, 64, 192)
  → Conv 3x3, s=(3,1), 256c → (1, 64, 256)   # collapse H to 1
  → Reshape → (64, 256)
```

Each conv is followed by BatchNorm + ReLU. Final output: 64 timesteps (plenty for 20-char labels), 256 channels.

**Why:** standard conv blocks (not depthwise-separable) are more expressive per-parameter for text. Asymmetric strides `(2,1)` collapse height while preserving W for timesteps — the OCR-specific trick MobileNetV3 doesn't do. Output channel count is ~6× what we had before.

### 2. Stack 2 BiLSTM layers

```
→ Dropout 0.2
→ BiLSTM(256) return_sequences=True
→ Dropout 0.2
→ BiLSTM(256) return_sequences=True
→ Dense(45)
```

### 3. Narrow the crop from 50% → 40% of card width

`BOTTOM_REGION_FRACTIONS` goes from `(0, 0.90, 0.50, 1.00)` → `(0, 0.90, 0.40, 1.00)`.

This excludes most of the artist-name bleed on modern cards without losing any collector info (which sits to the left of the artist credit). Quick, safe win.

## Expected outcome

| | v1 | v1.1 | **v2** (projected) |
|---|---|---|---|
| Params | 115K | 633K | **~3M** |
| Stem | MobileNetV3-Small | MobileNetV3-Small | **Custom OCR CNN** |
| BiLSTM | 1 × 96 | 1 × 256 | **2 × 256** |
| Crop width | 50% | 50% | **40%** |
| Train time on T4 | — | 1.5 h | **~3 h** |
| Plateau CER | 0.43 | 0.47 | **0.10–0.20** (goal) |

Target: **val_cer < 0.15 by epoch 30** — the threshold where Moxify's fuzzy-matching pipeline becomes reliable.

## What could still go wrong

- **CTC still tricky with deeper stem.** If the time dimension collapse timing is off, CTC could still mode-collapse. Safer to verify T=64 with a shape check before training.
- **3M params might be big for TFLite on older Android.** After training we benchmark inference latency. If it's >50ms/scan on a Nord-class device, we quantize harder or shrink LSTM units.
- **Still not enough data.** If v2 also plateaus >0.15, the next lever is data — 26k cards may just not be enough and we'd need to ingest more (unfilter the frame rule, or data-augment harder).

## Risk-ranked iteration if v2 plateaus

1. **Increase augmentation strength** (cheap, quick)
2. **Add attention on top of BiLSTM** (medium complexity)
3. **Expand dataset** via frame filter loosening OR older-frame retraining (moderate effort)
4. **Switch architecture to Transformer encoder** (big rewrite, reserved for if CTC-CRNN plateaus systemically)

## Implementation steps

1. Rewrite `src/moxify_ocr/models/crnn.py` with custom stem + stacked BiLSTM
2. Update `BOTTOM_REGION_FRACTIONS` in `src/moxify_ocr/data/crop.py` to `(0, 0.90, 0.40, 1.00)`
3. Update `tests/models/test_crnn.py` expected param count (~3M) and output shape
4. Bump `bottom_region.py` version string + config comment
5. Local forward-pass check (shape + save/load)
6. Push to `training-data` branch
7. User clones + Save Version → Save & Run All (Commit) on Kaggle

Estimated coder time: ~30 min. Estimated Kaggle wall time: ~3 hours.
